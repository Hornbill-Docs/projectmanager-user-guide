# Project Scope Changes
Projects can change in scope over the course of their life cycle. Recording the impact these changes can have is an important step.

### Adding scope changes to a project

Use the **+** icon to add a new project scope change

* **Scope Change**: The name/title of the project scope change
* **Description**: A description of the project scope change
* **Reason for Change**: The reason the project scope change exists
* **Approval Status**: The approval status of the project scope change
* **Approved By**: Select the user who approved the project scope change
* **Approval Date**: Select the date the project scope change was approved
* **Requested By**: Select the user who has requested the project scope change
* **Requested For**: Add the date the project scope change is requested by
* **Implemented**: Record whether or not the project scope change has been implemented
* **Scope Impact**: Describe the impact the project scope change will have on the project
* **Cost Impact**: Record the cost the project scope change will have on the project
* **Days Impact**: Record the extra time in days the project scope change will have on the project

  
Use the **Add** button to add the project scope change

### Managing scope changes on a project

Edit project scope changes by clicking on the project scope change you wish to edit from the list view

* **Scope Change**: Update the title of the project scope change
* **Description**: Update the project scope change description
* **Reason for Change**: Update the reason for the project scope change
* **Approval Status**: Update the approval status of the project scope change
* **Approved By**: Update the approver of the project scope change
* **Approval Date**: Update the date the project scope change was approved
* **Requested For**: Update the requester of the project scope change
* **Implemented**: Update the implementation status of the project scope change
* **Scope Impact**: Update the impact the project scope change will have on the project
* **Cost Impact**: Update the cost the project scope change will have on the project
* **Days Impact**: Update the extra time in days the project scope change will have on the project

### List View

View all project scope changes in list view.

* **Scope Change**: The name/title of the project scope change
* **Approval Status**: The approval status of the project scope change
* **Approved By**: The user who approved the project scope change
* **Date Approved**: The date the project scope change was approved
* **Cost Impact**: The impact in cost the scope change will have on the project
* **Implemented**: The project scope change implementation status
* **Created By**: Who added the project scope change
* **Create Date**: The date the project scope change was added
* **Day(s) Impact**: The impact in day(s) the scope change will have on the project
* **Last Modified By**: The user that last modified the project scope change
* **Last Modified Date**: The date the project scope change was last modified
* **Requested By**: The user that requested the project scope change

  
* Click on the edit icon for each project scope change to review, or edit it's properties.
* Use the search option to find project scope changes by their name

### Administration

When project scope changes are added, edited or removed this is audited and written to the audit trail tab of the overall project as well as against the audit tab on each scope change, where it can reviewed
